========================================
USING THE NEW FEATURES
========================================

CREATE A PROJECT:
   1. Click "New Project" button
   2. Enter name and description
   3. Click "Create Project"

OPEN A PROJECT (NEW!):
   1. Click any project card
   2. Opens in NEW WINDOW/TAB
   3. Work on multiple projects at once!

ADD PROJECT DETAILS (NEW!):
   1. In project page, find "Project Details" section
   2. Click "Edit" button
   3. Add context, links, credentials, notes
   4. Click "Save"

ADD A BUG:
   1. Click "Add Bug" button
   2. Fill in the fields:
      - Bug ID: Auto-generated (TP-1, TP-2...)
      - Portal: Admin Panel / Customer Side
      - Priority: Minor / Medium / Major / Critical
      - Module/Feature: Where bug was found
      - Bug Description: Detailed explanation
      - Status: Open / In Progress / Ready for QA / etc.
      - Assigned To: Developer / Frontend / Backend
      - Bug Link: URL or location
      - Client Notes: Client feedback
      - Developer Notes: Technical notes
   3. Click "Create Bug"

EDIT/DELETE BUGS:
   - Click "Edit" to modify a bug
   - Click "Delete" to remove a bug

========================================
NEW FIELDS EXPLAINED
========================================

BUG ID: Auto-generated (TP-1, TP-2, TP-3...)
   - Unique across all projects
   - Sequential numbering
   - No manual input needed

PORTAL: Where the bug appears
   - Admin Panel (backend/admin)
   - Customer Side (frontend/user-facing)

PRIORITY: How urgent the bug is
   - Critical: Breaks core functionality
   - Major: Significant impact
   - Medium: Moderate impact
   - Minor: Low impact

MODULE/FEATURE: Location of bug
   - Example: "Login Page", "Dashboard", "Checkout"

STATUS: Current state of bug
   - Open: Newly reported
   - In Progress: Being worked on
   - Ready for QA: Ready for testing
   - Closed: Fixed and verified
   - Reopened: Bug reappeared
   - Not a Bug: Works as intended
   - Needs Clarification: More info needed

ASSIGNED TO: Who's working on it
   - Developer: General developer
   - Frontend: Frontend specialist
   - Backend: Backend specialist
